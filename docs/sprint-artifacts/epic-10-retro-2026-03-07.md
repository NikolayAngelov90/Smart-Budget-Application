## What Went Well

### 1. i18n Architecture Excellence

The non-routing, cookie-based i18n approach using `next-intl` (Story 10-1) was an architectural win. By avoiding URL path prefixes (`/en/`, `/bg/`), the team preserved existing routing while enabling full internationalization. The `useTranslations(namespace)` hook pattern with namespaced JSON files became the standard for all subsequent stories.

**Impact:** Clean separation of concerns, minimal disruption to existing components, and a pattern that scales to additional languages without code changes.

### 2. Multi-Tier Caching Pattern

Story 10-5's exchange rate integration established a robust Redis (Upstash) primary + Supabase persistent fallback caching pattern with 1-hour TTL. This pattern, building on Epic 9's Redis migration work, demonstrates mature infrastructure thinking.

**Impact:** Reliable exchange rate data with minimal API calls, graceful degradation when Redis is unavailable.

### 3. Integration Test Gap Finally Closed

The CRITICAL integration test gap, flagged since Epic 6 (3 epics ago), was systematically addressed in Story 10-9. Using the test utilities library from Epic 9 (Story 9-2), the team established clear integration test patterns: `@jest-environment node`, mock `next/server` before imports, chainable Supabase mocks.

**Impact:** 35+ integration tests covering authentication, transaction CRUD, categories, dashboard, insights, and exports. Integration test guide documented for future reference.

### 4. Mobile-First Design Patterns

Stories 10-7 and 10-8 delivered production-grade mobile experience with elegant responsive patterns. The `useBreakpointValue` approach for mobile/desktop divergence (Drawer vs Modal, bottom nav vs sidebar) keeps a single codebase while providing native-feeling experiences.

**Impact:** Bottom navigation, swipe gestures, pull-to-refresh, haptic feedback, and smart PWA install prompt create a polished mobile experience.

### 5. CI/CD Pipeline Maturity

Story 10-10 wired existing deployment scripts into the CI workflow, adding env-check, pre-deployment-check, and Vercel preview URL steps. Combined with non-null assertion elimination, the pipeline now catches issues before they reach production.

**Impact:** Automated quality gates that prevent broken deployments; zero remaining `!` assertions in source code.

### 6. Test Suite Discipline

The test suite grew monotonically from 621 to 808 tests across the epic, with 100% pass rate maintained throughout. Every story added tests proportional to its scope.

**Impact:** 808 tests all passing with zero TypeScript errors and zero ESLint warnings.

### 7. Retro Action Item Follow-Through Improvement

75% follow-through on Epic 9 action items, up from 59% in the previous cycle. All CRITICAL and HIGH items completed. This demonstrates the retrospective process is working and the team is building accountability muscle.

## What Didn't Go Well

### 1. Incomplete i18n Coverage

Stories 10-1 and 10-2 left several components not fully i18n-ized: ConfirmDeleteModal, PaginationControls, some Insights components, and portions of Dashboard, Transactions, and Categories pages. These were noted but not formally tracked as technical debt.

**Root Cause:** Scope management - the stories focused on framework setup and core translations rather than exhaustive coverage. Components with complex dynamic strings were deferred.

**Recommendation:** Create a follow-up task to complete i18n coverage for remaining components.

### 2. Incomplete Documentation in Story 10-2

Story 10-2 (Bulgarian translations) has "TBD" in the Dev Agent completion notes despite being marked done. This suggests the documentation was rushed during completion.

**Root Cause:** Process gap - completion notes weren't verified as part of the "done" checklist.

**Recommendation:** Add documentation completeness to the story completion checklist.

### 3. Virtualization Skipped

Story 10-8 skipped transaction list virtualization because pagination makes it unnecessary at current scale. However, `@tanstack/react-virtual` was mentioned in the spec but never installed.

**Root Cause:** Pragmatic decision - pagination handles the current data volume. Not truly a problem, but the spec-to-implementation deviation wasn't formally documented.

### 4. Deferred Analytics Dashboard

The analytics visualization dashboard (epic9-medium-3) was carried forward but never addressed. Two epics of analytics data collection (Stories 9-4, 9-5) without any visualization means the data isn't being used.

**Root Cause:** Feature priorities (i18n, multi-currency, mobile) took precedence. Understandable, but the analytics investment loses value without visualization.

### 5. ESM/Jest Compatibility Friction

Story 10-1 hit ESM module resolution issues with `next-intl/server` in Jest. This is a recurring pattern across epics - Jest's CommonJS-first approach clashes with modern ESM packages.

**Root Cause:** Ecosystem-level issue. Jest's ESM support remains experimental.

**Workaround:** Shared `config.ts` constants that avoid importing ESM-only modules in test contexts.

## Key Insights and Learnings

1. **Cookie-based i18n scales better than URL-based for existing apps.** The non-routing approach avoided breaking changes to the entire routing structure. Future i18n implementations should consider this pattern first.

2. **Multi-tier caching (Redis + persistent fallback) is the right pattern for external API data.** Exchange rates, and potentially other external data, benefit from fast cache (Redis) with reliable fallback (Supabase).

3. **Integration tests are achievable with good utilities.** The test utilities library from Epic 9 removed the barrier. The chainable mock pattern makes API route testing straightforward.

4. **Responsive divergence via `useBreakpointValue` keeps code maintainable.** Using Chakra's built-in responsive utilities (Drawer vs Modal, bottom nav visibility) avoids the complexity of separate mobile/desktop codebases.

5. **Action item tracking works when tied to stories.** Mapping retro action items directly to story numbers creates clear accountability. The 75% follow-through rate validates this approach.

6. **20% infrastructure policy is sustainable.** Epic 10 hit exactly 20% (2/10 stories) and delivered meaningful infrastructure work (integration tests + CI/CD) without slowing feature delivery.
